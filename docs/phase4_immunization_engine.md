## Phase 4 Immunization Engine

## Purpose

Phase 4 defines how immunization-gap analysis should work for the real patient
record and what limitations need to be communicated clearly.

## Current Questions

- Is `Immunization` available and complete enough in the current FHIR response?
- If not, what is the safest fallback explanation for the demo?
- Which accredited recommendation source should anchor the comparison logic?

## Expected Outputs

- a chosen recommendation source
- comparison rules that reflect age, history, timing, and missing data
- a clear statement of limitations and non-clinical disclaimers
